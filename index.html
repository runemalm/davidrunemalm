OpenDDD.NET

Several years ago now, I came across the concept of Domain-Driven Design. I was heavily inspired, since it provides a holistic approach to software development. By focusing on the domain model, complex domains can be understood and implemented in software. This often leads to simple solutions that are easy to understand, maintain and scale. As part of my learning, and by the personal need for a DDD framework to aid in development efforts moving forward in my career, I started this open-source project. The focus of this project is to implement a roadmap of DDD building blocks piece by piece until reaching a framework that can be used to fully implement software using key DDD principles and building blocks.

  • ✅ Following key DDD principles and building blocks.
  • ✅ Clear roadmap to a comprehensive framwork.
  • ✅ Useful tools such as repository, entity, etc.

Open Banking Proof-of-Concept Client

This proof-of-concept client for open banking services was inspired by my experience working in the fintech industry. It’s a drop-in client for websites, enabling seamless bank integrations via a backend. The architecture allows scalability, with developers able to work independently on new bank integrations using simple subclassing techniques. While built as a monolith, it's designed with scalability in mind, allowing for scaling by breaking out bank integrations into adapter microservices when need arises.

  • ✅ Simple monolithic architecture.
  • ✅ Designed for scalability.
  • ✅ One developer per bank integration.
  • ✅ Javascript SDK for simple website integration.
